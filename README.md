# banking-transaction-analysis
Banking transaction data analysis using SQL, Python, and Power BI dashboard.


# Banking Transaction Data Analysis

## Project Overview

This project focuses on analyzing banking transaction data to identify customer spending patterns, transaction trends, and revenue distribution across different locations. The objective of the analysis is to transform raw transaction data into meaningful insights using data analysis and visualization techniques.

The project demonstrates practical data analysis skills including **data cleaning, SQL querying, exploratory data analysis using Python, and dashboard creation using Power BI**. By combining these tools, the project highlights how financial transaction data can be analyzed to understand customer behavior and transaction activity.

## Tools and Technologies

The following tools and technologies were used in this project:

* **SQL (SQL Server)** for querying and aggregating transaction data
* **Python** for data cleaning and exploratory data analysis
* **Pandas** for data manipulation
* **Matplotlib and Seaborn** for data visualization
* **Power BI** for building an interactive dashboard
* **Jupyter Notebook** for performing the Python analysis

## Dataset

The dataset used in this project contains banking transaction records with information related to customer transactions. Important attributes in the dataset include:

* Customer ID
* Customer Gender
* Customer Location
* Transaction Amount
* Transaction Date

A sample version of the dataset is included in the repository due to GitHub file size limitations.

## Data Analysis

The analysis was conducted in multiple stages to better understand the transaction data.

### SQL Analysis

SQL queries were used to extract useful insights from the dataset, including:

* Total number of transactions
* Total transaction value
* Average transaction amount
* Number of unique customers
* Gender-wise transaction analysis
* Location-wise transaction revenue
* Highest and lowest spending customers
* Monthly revenue trends
* Top cities generating the highest transaction value

These queries helped summarize the dataset and identify patterns within the transaction records.

### Python Analysis

Python was used to perform exploratory data analysis and visualize important trends in the dataset. Data cleaning and preprocessing were performed to ensure data consistency before analysis.

Several visualizations were created to understand the distribution of transaction amounts and analyze transaction patterns across different customer groups.

## Dashboard Visualization

A Power BI dashboard was created to present the key insights obtained from the analysis in a visual format. The dashboard includes interactive visualizations that help in understanding transaction trends and customer activity.

The dashboard highlights:

* Total transaction amount
* Monthly transaction trends
* Gender-wise transaction distribution
* Location-wise revenue contribution

## Key Insights

The analysis of the banking transaction dataset revealed several important insights:

* Male customers contributed the majority of the total transaction value.
* A small portion of transactions contained missing gender information.
* Certain locations generated significantly higher transaction revenue.
* Transaction activity varied across different months.

## Conclusion

This project demonstrates how banking transaction data can be analyzed using SQL, Python, and Power BI to generate meaningful insights. By applying data analysis techniques, it becomes possible to understand transaction behavior, identify financial trends, and support data-driven decision-making.
